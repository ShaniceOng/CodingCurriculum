Objects and Classes

Inheritance

When class A inherits from class B, class A acquires the properties of class B.

The subclass is the class that inherits the properties of another class.

The superclass is the class whose properties are inherited by another class.

Extends

We use the extends keyword to inherit the properties of a class.

Below is an example of how the extends keyword is used (where Super is the name of the superclass and Sub is the name of the subclass). 



class Super {
//instance variables and methods here
}

class Sub extends Super {
//instance variables and methods here
}

When should extends be used?

When the two classes have an IS-A relationship.

Hover over the Student class to see what happens when you use “extends” on the Student class to inherit the Person class. In this example, we would instantiate a new object of the Student class like we normally do

Inheritance is useful as it reduces data redundancy, making our code neater and more concise.

Contructors In Inheritance

A subclass can call a constructor defined by its superclass by using the following form of super: 


super(arg-list);

where arg-list specifies any arguments needed from the constructor in the superclass. super() must always be the first executed in a subclass’ constructor.

Overriding

Overriding allows a subclass to implement a superclass’ method based on its requirements. It is the process of creating a modified version of an existing method. 


class Animal 
{
    public void saySomething()
    {
        System.out.println("I can talk!");
    }
}

class Cat extends Animal 
{
    public void saySomething()
    {
        System.out.println("Meow!");
    }
}

class Dog extends Animal 
{
    public void saySomething()
    {
        System.out.println("Bark!");
    }
}

public class TestOverriding
{
    public static void main (String[] args)
    {
        Animal animal = new Animal();
        Animal cat = new Cat();
        Dog dog = new Dog();

        animal.saySomething(); 
        cat.saySomething(); 
        dog.saySomething();
    }
}

Terminal Output


          I can talk!
          Meow!
          Bark!

In this example, we can see that both the Cat and Dog class inherit from the Animal class. However, both their saySomething() methods have been overridden to fit the specific requirements of each animal. This allows the objects “animal”, “cat” and “dog” to call their own versions of saySomething(). Notice that, although “cat” is declared with reference type Animal, the method that belongs to the Cat object is run when called on cat. This is because “cat” is declared as a Cat object and because the Animal class recognizes the saySomething() method too.

Overriding is very useful when many classes are inheriting from a superclass and a few of them require modifications to their properties. You could override those that need overriding instead of re-writing the method for every class.

IMPORTANT! Rules to follow when overriding methods:

  1. Its return type should be the same or a subtype of the return type of the original method.
  2. If the superclass method is public, this class’ method stays public.
  3. If a method has been declared final, you can’t override it. No buts!
  4. If both superclass and subclass belong to the same package, the subclass can override any of the superclass’ non-private, non-final methods.
  5. A subclass can override methods of a superclass from a different package if they are non-final and either public or protected.
  6. You can only override a method if it is inherited by that subclass.

Super Overriding

The super keyword allows us to have methods and attributes of the same name in the superclass and subclass and still call them as different attributes and methods. 


public class Super {
    String name = "superclass";

    public void describe() {
        System.out.println("This is the super class’ describe method");
    }
}

public class Subclass extends Super {
    String name = "subclass";
    String superName = super.name;

    public void describe() 
    {
      System.out.println("This is the subclass’ describe method");
    }

    public void superDescribe()
    {
      super.describe();
    }

    public static void main (String[] args) 
    {
      Subclass subclass = new Subclass();
      String x = subclass.name; 
      System.out.println(x);
      String y = subclass.superName;
      System.out.println(y);
      subclass.describe();
      subclass.superDescribe();
    }
}

Terminal Output


          subclass
          superclass
          This is the subclass’ describe method
          This is the super class’ describe method

Interfaces

Interfaces are similar to classes but contain abstract methods and are, themselves, implicitly abstract.

Describes attributes and behaviors of an object Contains behaviors that a class implements
CAN be instantiated CANNOT be instantiated

Class extends extends Class Class implements interface
Class can extend one clas Interface can extend multiple interfaces
Class can implement multiple interfaces 


public interface Halogen 
{
  public void reactAlkaliMetal (ChemicalElement alkaliMetal);
}

To implement interfaces, use the implements keyword, like we did before. Observe that interface methods cannot have method bodies. When a class implements an interface, it is signing a contract to implement every method in that interface. The consequence of breaking this contract is that the program won’t compile! 


public class Nitrogen implements Halogen 
{
  public void reactAlkaliMetal (ChemicalElement element)
  {
     //method body 
  }
}

Did you know?

The method to extend other interfaces from an interface is similar. The only additional rule is that when one interface extends multiple interfaces, these interfaces are declared after the “extends” keyword and are separated by commas. 


public interface IndustrialRevolution extends HistoricalEvent, Industrialisation 
{
  //methods
}

Polymorphism

A polymorphic object has the ability to take many forms. How do we know if an object is polymorphic? Objects are polymorphic if they have IS-A relationships with more than one class. We can say that all objects in Java are polymorphic, since they have an IS-A relationship with their own type and with the Object class.

Every object has to be declared with a reference variable. This reference variable cannot be changed once assigned.

NOTE: Although we can, for example, change a variable “x” from integer to string using the Integer.toString(x) method, we are not actually changing the reference variable of x, but making a String copy of x.

Since polymorphic objects can take many types, we can declare them with the reference variable of any of the classes it inherits, as shown in the example below. 


public interface Halogen 
{
  //abstract methods
}

public class ChemicalElement 
{
  //class body
}

public class Nitrogen extends ChemicalElement implements Halogen 
{
  //class body
}

Here, Nitrogen is polymorphic because it inherits three classes, Halogen, ChemicalElement and Object.

Since Nitrogen is polymorphic, we can do this 


public static void main(String[] args)
