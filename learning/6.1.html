Objects and Classes

Classes -> Objects

As Java is an Object-Oriented programming language, learning to work with objects and classes is essential to improving your Java skills.

Objects have states/attributes and behaviors/methods

  • A DOG has a name, breed, age and barks, eats, walks
  • A STUDENT has a name, e-mail address, school and studies, writes, submits homework.

Classes are the “cookie cutters” or “blue prints” with which objects are produced.

Creation

Create A Class

This is an example of the basic structure of a class


public class Computer 
{
  String name;
  int yearCreated;
  int massInKg;

  public Computer (String name, int year, int mass) 
  {
    this.name = name;
    this.yearCreated = year;
    this.massInKg = mass;
  }

  void interpretInput() 
  {
    int count;
    //method body
  }

  void calculate() 
  {
    //method body
  }
}

First, we have to give this class some attributes. These variables will be initialized when an object of this class is created.


String name;
int yearCreated;
int massInKg;

These variables are called instance variables. They are declared within the class but not inside any methods or constructors and can be accessed from anywhere within this class.

Now we need a way to initialize new objects of this class. Cue the constructors!


public Computer (String name, int yearCreated, int mass) 
{
  this.name = name;
  this.yearCreated = yearCreated;
  massInKg = mass;  
}

This is a constructor. It takes parameters to create an instance of the class. Constructors have no return types and should have the same name as the class. They initialize the instance data of this object using the values given to them. Since each of the arguments to the constructor shadows one of the object’s fields, we use “this.” to initialize that value. For example, “this.name” is the instance data and it is equal to the address coming in from the calling program, “name”.

In this example, interpretInput() and calculate() are methods. These are the actions that objects of this class can carry out. Variables declared within methods, like “count”, are called local variables.

Create An Object

All objects are instances of the class they belong to. In order to create a new object, we must instantiate a class using new and a constructor from the class 


Computer eniac = new Computer(“ENIAC”, 1946, 27215);

In this line, we declare a variable, eniac, of object type Computer, instantiate the object with “new” and initialize the object with the parameters inside the parentheses.

Overloading

Overloading Methods

We can define two or more methods with the same name, as long as their parameter declarations are different. When we invoke an overloaded method, Java uses the types and/or number of arguments to determine which version of the method to call.

Overloaded methods MUST differ in type/number of parameters. 


public class OverloadMethods
{
  void drawShape()
  {
      System.out.println("Not a shape");
  }
  void drawShape(int x, int y, int z)
  {
      System.out.println("Draw a triangle of perimeter " + (x+y+z));
  }

  void drawShape(int w, int  x, int y, int z)
  {
      System.out.println("Draw a rectangle of perimeter " + (w+x+y+z));
  }
}

public class Overload {
  public static void main (String[] args)
  {
      OverloadMethods overload = new OverloadMethods();
      overload.drawShape();
      overload.drawShape(2,3,2);
      overload.drawShape(5,10,5,10);
  }
}

Terminal Output


          Not a shape
          Draw a triangle of perimeter 7
          Draw a rectangle of perimeter 30

As we can see above, the drawShape method is overloaded three times, once without parameters, once with three parameters and once with four parameters. Java looks for a match between the parameters used to call the method then calls the matching method.

Overloading Constructors

In a similar way as overloading methods, you can overload constructors. 


public class History{
  String historicalEvent;
  String location;
  String startDate;
  String endDate;

  History (String event, String place, String start, String end)
  {
      historicalEvent = event;
      location = place;
      startDate = start;
      endDate = end;
      System.out.println(historicalEvent + " took place in " + location + " between " + startDate + " and " + endDate);
  }

  History (String event, String place, String start)
  {
      historicalEvent = event;
      location = place;
      startDate = start;
      endDate = "-";
      System.out.println(historicalEvent + " took place in " + location + " and started in " + startDate);
  }

  History (String event, String place)
  {
      historicalEvent = event;
      location = place;
      startDate = "unknown";
      endDate = "unknown";
      System.out.println(historicalEvent + " took place in " + location);
  }

}

public class OverloadConstructor {
  public static void main(String[] args) {
      History event1 = new History("The English Civil War", "Kingdom of England", "1642", "1651");
      History event2 = new History("Battle of Waterloo", "Waterloo", "1815");
      History event3 = new History("Battle of Naseby", "Northamptonshire");
  }
}

Terminal Output


The English Civil War took place in Kingdom of England between 1642 and 1651
Battle of Waterloo took place in Waterloo and started in 1815
Battle of Naseby took place in Northamptonshire

An overloaded constructor is called by Java when it checks the parameters specified during instantiation of the object and matches it to an overloaded constructor in the class. Therefore, this program compiles without errors although we use different numbers of parameters to initialize objects of the same type.


Encapsulation

Encapsulation forms a protective barrier to prevent outside access. With encapsulation, you can hide instance data definitions and protect the integrity of instance data. We can enforce data integrity by defining all instance variables in the class as private.

To perform encapsulation

  1. Declare the class variables as private.
  2. Make public setter methods, which allow other classes to set the value of the variables without giving them direct access to the variables.
  3. Make public getter methods, which allow other classes to get instance values for use outside the class.

Our goal: To make as few variables and methods public as possible! If other classes do not need to access them, they should be defined as private.

Do you remember that Person class we looked at in the Inheritance section? We are going to create a similar class but, this time, we will use encapsulation. 


public class Person {
  private String name;
  private int age;
  private String email;

  private void sayHello()
  {
    System.out.println("Hello, it's me!");
  }

  public String getName()
  {
    return name;
  }

  public int getAge()
  {
    return age;
  }

  public void setName (String name)
  {
    this.name = name;
  }

  public void setAge (int age)
  {
    this.age = age;
  }

  public void nameAndAge ()
  {
    sayHello();
    System.out.println("My name is " + name + " and I am " + age);
  }
}

public class TestEncapsulation {
  public static void main (String[] args)
  {
    Person adele = new Person();
    adele.setName("Adele");
    adele.setAge(28);

    adele.getName(); //returns "Adele"
    adele.getAge(); // returns 28
    adele.nameAndAge();
  }
}

Terminal Output


          Hello, it's me!
          My name is Adele and I am 28

Notice that the method sayHello() is also private because we only need to use it within the Person class.

Why is encapsulation so important?

  • To avoid misuse of classes and objects.
  • Gives the class more control over the variables stored in its fields.
  • A class can change the data type of a field and without other classes having to change any of their code.

Abstraction

Abstraction is used to separate the essential from the full detail of the class.

Abstract Classes

Abstraction hides implementation details from the user, displaying only functionalities. It provides a selective and simplified view of the concepts being represented as objects.

Properties of Abstract Classes
IS DECLARED... Using the "abstract" keyword like this:
public abstract class Scientist { //class body }
MAY OR MAY NOT... Contain abstract methods
- abstract methods are methods without a body of functions.
CANNOT BE... Instantiated
CAN BE... Inherited by a concrete class by using the same “extends” keyword and method as with concrete classes.
HAS TO BE... used

Abstract Methods

Abstract Methods are used if you want a class to have a method but you want its subclasses to determine the implementation of this method.

Properties of Abstract Methods
IS DECLARED... Using the “abstract” keyword like this:
public abstract void setDifference();
DOES NOT... Contain a method body (only contains a method signature)
HAS... A semicolon at the end of declaration, instead of curly braces
MUST BE... Declared within an abstract class
MUST BE... Overridden by a concrete class that inherits the abstract class containing this abstract method
