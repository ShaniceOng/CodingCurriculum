Scope

Scope is a very important idea in programming, it is the idea of where and how you can access different parts of a program from. For beginners you may look at piece of code and see variables declared in different places however they all have different values and it can be confusing as to how it works.

For example you might see a i = 0 in one part of the program and a i = 1 somewhere else but both have different values and are used differently. So how does this work? To understand this you need to understand the different kind of scopes.

Class Level Scope

Class level scope means that a variable or a method is limited to a specific class.When a variable is created within a class, the scope of the variable is said to be "local" to the specific class. This means that the variable is created when the specific class is activated in a program, and when the class is no longer used, the variable is no longer valid, therefore it does not take up any memory space. This variable is usually declared at the top of the class as a convention.Take the short program below as an example: 



public class Car
{
  String name;
}


public class cycle
{
  string name;
}

here even though the two variable names are the same, they can be used at the same time because their variables are local to their own specific classes. So they have seperate memory addresses once each class is called, and they can be used at the same time.

Method Scope

Just like a variable is specific, a variable can also be specific to a method. When a variable is created inside a method, that variable is said to be "local" to the method. So when the method is called by the program, the variable is created and assigned some memory. When the method is no longer needed, the variable isn't needed as well, and there is no way to access that certain variable. Again, similar to variables created within a class, the variables are declared at the top of a method. Take an example below:



public int getName()
{
  Int length;
}




public double getAddress()
{
  Int length;
}

In both these methods, the variables are declared at the top of the method, and they have the same name. However, both can be used at the same time because when each method is called, they are allocated seperate memory spaces, and they can only accessed by their specific method. So the getName can access it's own length variable and the getAddress can access its own length variable. But when the method is no longer called, the variable is not valid, and other methods cannot access these variables.

Loop Scope


Variables are usually created within loops. Amongst a few othere reasons, these variables help keep track of how many times the loop is run. When the variable is created within the loop, it is said to be "local" to the loop: What this means is that the variable can only be accessed by the loop, and it is created when the loop is called on. Thus, when the loop is no longer running, the variable does not exist, and it is not assigned computer memory space. For example: 


for(int i = 0; i < 10; i++)
{
  System.out.println("hello");
}

The variable i is created in the loop above. Therefore, only the loop can access this variable, and it is created when the loop is called. When the loop is not running, the variable i for this certain loop, does not exist, therefor it cannot be accessed by the loop and any other method.
