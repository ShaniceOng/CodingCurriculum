Recursion

Just like loops, if statements and other blocks of program code, recursion is a way of solving a certain problem. It is formally defined as a method that calls itself; this means that within the method body, there is a call to the method itself. This sounds really weird, doesn't it? Here's an example to clear it all up: 


  public int fact(int n)
  {
    if(n == 1)
    {
      return 1;
    }

    else
    {
      return n * fact(n-1);

    }

  }

You may not be familiar with the factorial function; this functions essentially calculates the product of a number and its predecessors. For example, the calculating the factorial of 5 would be 5*4*3*2*1 which equals to 120. Now we'll finally explain how the magical recursion will work. The entire point of recursions is that the entire problem is broken down into smaller problem. This is a good way to solve problems, and it also makes programming easy. This is because some tasks take a lot of time and effort to write; recursion reduces this to just a couple lines of source code in java.



Let's take an actual example of the factorial function works. So if we try call fact(4) in the main function of the class, here is how the computer will evaluate it: the if statement is skipped because n does not equal 1. Therefore it returns 4 * fact(3). Since fact(3) is called within the method, the factorial function is called again. Following the same process, it will return 3 * fact(2). So here's what it will look like:

fact(4) = 4 * fact(3)

fact(3) = 3 * fact(2)

fact(2) = 2 * fact(1)

fact(1) = 1

The last one will go through the if statement and the value of n will be equivalent to 1. Hence the answer will be 4*3*2*1 = 24.

This is where we talk about general principles of recursion. Each recursive method needs to have what is called the base case; in this block of code, the base case is in the if statement. Remember we talked about recursion breaking a problem into smaller problems? The base case is the smallest solution, and it has to be an actual value. This is because the recursive method will keep calling itself, and the program will eventually crash.



When designing recursive methods, you need to keep the following facts in mind

  • You need to define a base case for every problem, this will be the smallest solution.
  • The recursive method should return a value
  • The recursive method should find a way of making the problem smaller.
  • There should be a call back to the problem
